데이터 분석 기획

	분석의 삼요소 data, analytic model, analyst -> 가치 창출

	실제 분석을 수행하기 에 앞서 수행할 과제의 정의 및 의도했던 결과를 도출 할 수 있도록 적	절하게 관리할 수 있는 방안을 사전에 계획

	분석 대상 및 방법에 따른 4가지 유형
	
	1. 개선을 통한 최적화 : 해결해야할 문제를 알고 분석 방법도 알고 있는 경우
	2. 솔루션을 찾아내는 방식 : 해결할 문제는 알지만 분석 방법을 모르는 경우
	3. 통찰 : 대상이 명확하게 무엇인지 모르는 경우, 기존 분석방법을 활용
	4. 발견 : 분석의 대상 자체를 새로 도출

	과제 중심적 -> 당면한 과제를 빠르게 해결
	장기적 마스터 플랜 -> 분석과제정의를 위해

	의미있는 분석 -> 분석 기술, it및 프로그래밍, 분석주제에 대한 도메인 전문성
			의사소통, 분석 대상 및 방식에 따른 다양한 주제를 과제 단위 혹은 마스터			플랜 단위로 도출 할 수 있어야 한다.
			프로젝트 관리 역량, 리더십 역량이 필요.

	분석 기획 - 수립과정 : 가용 데이터, 적절한 유스케이스, 분석과제 수행을 위한 장애요소
	
	분석을 위한 데이터의 확보. -> 데이터 유형에 따라 적용가능한 솔루션 및 분석방법
	정형, 비정형, 반정형

	기존의 유사 분석 시나리오 및 솔루션의 활용

일반적 방법론 -> 계층적 프로세스 모델.
	단계(각 단계별 산출물) -> 테스크(각 단계를 구성하는 단위활동) -> 스텝 (입력자료, 처리 및 	도그. 출력자료)

KDD 분석 방법론 -> 데이터베이스에서 의미있는 지식을 탐색하는 데이터 마이닝 부터.
	-> 데이터 패턴을 찾는 9개의 프로세스
	1. 분석대상 비즈니스 도메인의 이해
	2. 분석대상 데이터셋 선택과 생성
	3. 데이터에 포함되어있는 노이즈와 이상값 들을 제거하는 정제작업 혹은 선처리
	4. 분석 목적에 맞는 변수를 찾고 필요시 데이터의 차원을 축소하는 데이터 변경
	5. 분석 목적에 맞는 데이터 마이닝 기법 선택 ->>>>>>>>
	6. 분석 목적에 맞는 데이터 마이닝 알고리즘 선택
	7. 데이터 마이닝 시행
	8. 결과에 대한 해석
	9. 지식 활용

	SELECTION -> PROCESSING -> TRANSFORMATION -> DATAMAING -> EVALUATION
	비즈니스 도메인에 대한 이해. 프로젝트 목표를 정확하게 설정(필요한 데이터의 설정, 추가적인 데이터셋 생성) -> 추출된 분석 대상용 데이터셋에 포함되어있는 이상값 및 잡음을 제거하고 의미있는 데이터로 정제작업. -> 분석 목적에 맞는 변수를 선택. 데이터 차원을 축소. -> 분석용 데이터 셋 이용. 분석 목적에 맞는 마이닝, 알고리즘 선택. -> 패턴을 찾거나 데이터 분류 예측. -> 해석과 평가. 분석 목적과의 일치성.

CRISP-DM 각 단계별로 피드백을 통해 단계별 완성도를 높이는 구조
	업부 이해 -> 데이터 이해 -> 데이터 전처리 -> 모델링 -> 평가 -> 전개


	1)비즈니스 이해 및 범위 설정 -> 비즈니스에 대한 이해. 내부업무 매뉴얼과 관련 자료, 외부의 관련 비즈니스 자료 조사를 통해 향후의 방향을 설정. 프로젝트의 범위를 명확하게 설정.

	2) 프로젝트 정의 및 계획 수릭 -> 모델 운영 이미지를 설계하고 모델 평가 기준을 설정. 프로젝트의 정의를 명확하게. 모든 프로젝트의 수행계획을 작성( 데이터 확보의 계획, 빅데이터 분석 방법, 일정게획, 예산 계획, 품질계획,인력구성계획,의사소통계획) 프로젝트의 목적 및 배경, 기대효과, 수행 방법, 일정 및 추진 조직, 프로젝트 관리 방안 작성

	3) 프로젝트 위험계획과 수립 -> 발생가능한 모든 위험을 발굴하여 사전에 대응방안을 수립(안정성 확보) 

	

데이터의 준비 -> 	필요 데이터의 정의하고 획득 방안을 수립
		데이터 스토어 설계 -> 구조화된 형식. 관계형 데이터베이스 사용
				효율적인 저장과 활용을 위해 논리적 물리적 설계를 구분하여 				설계
		데이터의 수집 및 정합성 점검 -> 구축된 데이터 스토어에서 데이터 수집.
		수집된 데이터는 품질 점검을 위해 정합성 점검을 실시하는 과정

데이터 분석
		분석용 데이터의 준비 -> 비즈니스 룰을 확인하고, 분석용 데이터 셋 준비

		텍스트 분석 -> 텍스트데이터 확인 및 추출 후 데이터 분석

		탐색적 분석 -> 다양한 데이터 시각화를 활용하여 가독성을 톺이고, 데이터 형상 및 			분포 등의 데이터 특성을 파악하는 테스크
			다양한 관점별로 기초 통계량 산출, 데이터의 시각화

		모델링 -> 분석용 데이터를 이용한 가설 설정을 통해 통계 모델을 만들거나 데이터			의 분류, 예측, 군집 들의 기능을 수행하는 모델을 만든다.
		과적합과 일반화를 위해 훈련용과 테스트용으로 나눈다.
		데이터분할 -> 데이터 모델링 -> 모델적용 및 운영방안 (알고리즘 설명서)-> 모델평		가 및 검증

시스템 구현
		알고리즘 설명서에 근거하여 실제 시스템을 구축하고 테스트를 통해 안전성을 확보, 		지속적으로 사용가능하도록 메뉴얼을 만들고 교육 실시

평가 및 전개
		모델링한 것을 계속 사용할 수 있도록 발전계획을 세우고 프로젝트 성과를 평가한 		귀 보고서를 작성하여 산출물 및 프로세스를 지식 자산화



분석 과제 발굴 -> 데이터를 활용하여 생각지 못하였던 인사이트 도출 + 상향식 접근 방식
	상향식 : 문제 정의 자체가 어려운 경우. 데이터를 기반으로 해결방안 탐색, 지속적 개선
	하향식 : 해결법을 찾기 위해 과정 체계적으로 수행
	-> 두 단계를 반복적으로 수행함에 따라 상호 보완하는 것이 분석의 가치를 높일 수 있는 최	적의 의사결정 방식

	하향식 : 문제 탐색 단계 -> 문제 정의 단계 -> 해결방안 탐색 단계 -> 타당성 검토
	상향식 : 기업에서 보유하고 있는 다양한 원천 데이터로 부터 분석을 통해 통찰력과 지식을 얻는 방법. -> 데이터 분석의 목적이 명확하지 않다. 데이터 자체의 결합, 연관성, 유사성 들을 중심으로 상태 표현 -> 인과관계로 부터 상관관계 분석으로 이동. 빅데이터 분석에서의 주요 변화. -> 왜 그런지 알아내는 것이 중요.

데이터 마이닝 기법 - 대규모로 저장된 데이터 안에서 체계적이고 자동적으로 통계적 규칙이나 패턴을 찾아 내는 것.

	1. 연관성 분석 - 일련의 데이터로 부터 연관 규칙을 발견하고 상호 연관성을 밝히는 것.
		-> 명확하게 관련이 있는지 없는지 알기 어려울 것 같다.
		-> 생각치 못한 연관성을 알아낼 수 있다.

	2. 군집 분석 - 이질적인 집단을 몇개의 동질적인 소집단으로 세분화 하는 작업.
		-> 사전에 정의된 집단을 가지고 있지 않다는 점. 여러 집단의 데이터들이 섞여 있고 		각 데이터 소속 집단을 모르는 경우 유사한 속성을 갖는 데이터의 군집을 찾는 분석 			방법.
		-> 주어진 데이터 중 유사한 것들만 몇몇의 집단으로 그룹화하여, 각 집단의 성격을 			파악, 데이터 전체의 구조에 대한 이해를 돕고자 하는 탐색적 데이터 마이닝 방법론			이다.
	
	3. 의사결정 나무 - 우리가 관심을 가지는 분류의 문제를 해결하기 위한 알고리즘
		-> 분류를 하기 위한 목표변수에 영향을 줄 수 있는 입력 변수들을 이용하여 최적의 		분류를 위한 의사결정 규칙을 생성하게 된다.	
		
		특정 타겟 변수에 의해 여러가지 성질의 데이터를 보다 유사한 성직의 소그룹으로 분류하거나 예측 이상치나 편향된 분포에 민감하지 않고, 결측치를 하나의 노드로 구분 가능하다는 장점
		
		뿌리 노드 - 의사결정 나무가 시작되는 전체의 데이터로 구성
		자식 노드 - 한가지 마디에서 2개 이상의 마디로 구분된 마디
		부모 노드 - 자식마디의 상위 마디
		잎 또는 끝 노드 - 각각 줄기의 최종에 위치한 마디. 끝마디의 갯수만큼 규칙이 생성
	
		뿌리에서 끝까지 각각의 노드를 구성하기 위한 분리기준을 설정하게 된다.
